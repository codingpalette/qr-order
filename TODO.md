## 기술 부채

| 항목 | 현재 상태 | 개선 방향 |
|------|----------|----------|
| i18n | 한국어 하드코딩 | Paraglide.js 또는 next-intl 도입 |
| 테스트 | 0% 커버리지 | Vitest + Playwright 도입 |
| 에러 처리 | 최소한의 try/catch | 전역 에러 바운더리 + API 에러 핸들러 |
| 이미지 | `<img>` 태그 직접 사용 | `next/image` 전환 (최적화/lazy) |
| 폼 검증 | 일부 Zod, 일부 수동 | React Hook Form + Zod 통일 |
| API 패턴 | 클라이언트 직접 Supabase 호출 | 민감 작업은 Server Actions/Route Handlers로 |
